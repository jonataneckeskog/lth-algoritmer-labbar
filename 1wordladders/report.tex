  \section{Results}

  The script did say that all of my problems were correct. I wrote separate unit tests for the code too before I ever ran the \texttt{check\_solution.sh} script, so it passed on the first try.
  On input 5 the program took about 15 seconds to run, while on input 6 it took $\sim$2 seconds. These are about the same size.
  What takes the longest is certainly the pathfinding algorithm. BFS has to check all vertices and edges in the worst case.

  \section{Implementation details}

  \todo[inline]{How did you implement the solution? Which data structures were used? Which modifications to these data structures were used? What is the overall running time? Why?}

  I implemented the data structure first. For this I used a \verb|HashMap<string, HashSet<string>>| (\texttt{Dictionary} in C\#).Essentially, it's a map between words and other words they connect to.
  For the pathfinding algorithm I used another \texttt{HashMap} and a \texttt{FifoQueue}; this was done to keep track of explored nodes, their distances, and nodes I was yet to explore.
  The time complexity is \textit{O(n+m)} where n is the number of nodes (words) and m is the number of edges (connections).
