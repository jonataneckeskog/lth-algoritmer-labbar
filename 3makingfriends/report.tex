  \section{Implementation details}

  I implemented the solution using Kruskal's algorithm. Essentially, the edges are first sorted based on length, shortest to longest.
  We can always know that if the current shortest edge connects a new verticy to the 'friendship graph' then we should include it.
  To keep track of the edges I used a simple array which I sorted in place, and I also included a simple array for looking up the
  'root' of any person based on their index (number). This allowed for easily checking if a person had been added.
  I do not really understand the question of 'overall running time'. For complexity it's O(M*logM) where M is the number of edges,
  but as for seconds it took about 3 second on my pc to run with every input.
